CSV-Format für die Batch-Prüfung

  • Eine Zeile pro Inhalt, Semikolon-getrennt (Komma wird als Fallback erkannt).
  • Spalte 1 = Node-ID (UUID, Pflicht). Spalte 2 = Repository (prod oder staging, optional — sonst Default).
  • Eine Kopfzeile node_id;repository ist erlaubt, aber nicht nötig; mit # beginnende Zeilen werden ignoriert.
node_id;repository
9a1b2c3d-1111-2222-3333-444455556666;prod
7f8e9d0c-aaaa-bbbb-cccc-ddddeeeeffff;staging

Ergebnis-Skala — was bedeutet welche Kategorie?

verified_user Unproblematisch
Positivnachweis einer freien/unkritischen Lizenz — Freigabe möglich (Attribution beachten).
help_center Nicht messbar
Kein belastbares Signal (leere Metadaten, keine Deklaration). Default-Deny: keine automatische Freigabe, aber auch kein konkreter Verdacht.
gpp_maybe Zu prüfen
Ein Warnhinweis liegt vor, ist aber nicht eindeutig (z. B. Agentur-Hinweis ohne Bildzuordnung, widersprüchliche Belege) — redaktionell klären.
gpp_bad Problematisch
Starkes Warnsignal ohne Gegenbeleg (z. B. Agentur-Nachweis auf der Fundseite) — nicht ausliefern.
