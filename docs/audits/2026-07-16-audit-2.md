# Code-Audit (2. Durchlauf) — Bild-Lizenz-Check

**Datum:** 2026-07-16 · **Anlass:** Vollpass nach Batch-System, Rate-Limiting,
SSRF-Guard · **Methode:** 3 parallele Lens-Audits (Security/Concurrency,
Correctness, Frontend), empirisch verifiziert · **Scanner:** ruff, npm audit (0),
madge (0 Zyklen), Prod-Build.

## 1. Executive Summary

Der zweite, tiefere Durchlauf fand einen **kritischen Correctness-Bug** (die
`normalize_license_uri`-Prüfung akzeptierte per Substring einen Tracking-Wrapper
als CC-Lizenz → **False GREIN**, d. h. ein geschütztes Bild würde freigegeben —
die schlimmste Richtung für Default-Deny) und einen **gebrochenen Prod-Build**
(`@angular/animations` fehlte). Beide sowie ~18 weitere Funde wurden **behoben und
per Regressionstest abgesichert**. Verbleibend und bewusst offen: DNS-Rebinding im
SSRF-Guard (deployment-mitigiert) und die fehlende App-Auth.

## 4. Behobene Findings (verifiziert)

### 🔴 Kritisch
- **False GREEN** ([wordlists.py](../../backend/app/wordlists.py) `normalize_license_uri`): prüfte den CC-Host per
  Substring → `…/r?u=creativecommons.org/licenses/by/4.0/` wurde als Lizenz
  normalisiert; hebelte auch „`plus:LicensorURL` immer RED" (c03) aus. **Fix:**
  echter Host + Pfad-Präfix. Regressionstest.
- **Prod-Build gebrochen**: `provideAnimationsAsync()` ohne `@angular/animations`.
  **Fix:** Paket installiert; Prod-Build sauber.

### 🟠 Hoch
- **RiskHub-Poisoning** ([risk_hub.py](../../backend/app/risk_hub.py)/[main.py](../../backend/app/main.py)): nicht-hex `phash` via `/api/risk-hub`
  ließ `_hamming_hex` werfen → SHA1/pHash-Abgleich für ALLE Bilder = ERROR
  (Denial of Detection). **Fix:** Hex-Validierung am Endpoint + defensives
  `_hamming_hex`.
- **CSV-Header** (batch.py `parse_csv`): nur physische Zeile 1 → das eigene
  Template (mit Kommentarzeilen) erzeugte eine Phantom-Node „node_id". **Fix:**
  Header an der ersten Inhaltszeile.
- **c08 reihenfolgeabhängig**: RED↔GREEN je nach DOM-Reihenfolge der Lizenz-Links
  (early `break`). **Fix:** alle Lizenzen klassifizieren, dann entscheiden
  (Agentur schlägt CC → deterministisch RED).
- **OOM**: unbegrenztes `file.read()` (Upload/CSV) + Body-Puffern trotz
  Größenlimit. **Fix:** begrenzte Reads (main.py) + **Streaming** mit hartem
  Byte-Cap in `net_guard.safe_get`.
- **Paginierung** (edu_sharing.py): Abbruch bei kurzer Seite trotz höherem `total`
  → Nodes übersprungen. **Fix:** bei bekanntem `total` nur danach abbrechen.
- **Frontend-Polling** (batch.component.ts): ein transienter Fehler beendete das
  Polling endgültig → Ergebnisse verloren. **Fix:** `catchError` im inneren
  Stream, erst nach 5 Fehlern aufgeben.

### 🟡 Mittel (behoben)
`" ap "`-False-RED entfernt · c01 keine erfundene Version · `_merge_fields` nach
Confidence · c09 exakter Basename-Match · c07 `maxlag`-Error → „nicht verfügbar" ·
Job-Feld-Mutationen + RiskHub-Reads unter Lock · `str(exc)` generalisiert (Log
statt Client) · Frontend: `fieldRows` als `computed()`, toter Router entfernt,
ungenutzte `MatChipsModule`/`MatCardModule` entfernt, Bundle-Budgets angehoben.
